网页开发基本规则

项目方案设计规范
1.项目设计要符合当前流行趋势，要符合用户的使用习惯；
2.项目设计要严格遵守用户需求，要符合用户需求；对于不确定的需求，要与用户进行沟通，确保需求的准确性；


编程规范
1.只允许用 html,css，JavaScript开发网页；
2.遵守基本的编程规范，代码要清晰，有注释，有说明；
3.只允许开发方便部署的静态网页，不允许开发动态网页；主页面index.html
4.代码命名要规范，变量名、函数名、类名等要具有描述性，释放缩写，可参考驼峰命名法；

数据库使用规范
重要！！！尽量避免使用数据库，如果必须使用数据库；
1.只允许使用sqlite本地数据库，不允许使用其他数据库；
2.数据库设计要符合当前流行趋势，要符合用户的使用习惯；
3.数据库设计要严格遵守用户需求，要符合用户需求；对于不确定的需求，要与用户进行沟通，确保需求的准确性；

UI设计规范
1.风格要统一，色调要协调，布局要合理，要符合用户的使用习惯；
2.样式要符合代码主题，同时要符合当前流行趋势；

UI界面设计基础原则

趋势导向： 设计必须符合当前 2024/2025 年的 Web 设计趋势（简洁、大圆角、微交互）。
用户至上： 严格遵循用户明确提出的需求。
主动确认： 对于模糊或不确定的需求，必须先列出“需求确认清单”与用户沟通，不得擅自臆造核心功能。

1.响应式布局 (Responsive Design)： 页面必须能够自适应不同屏幕尺寸（Mobile, Tablet, Desktop）。推荐使用 CSS Flexbox 或 Grid 布局，避免绝对定位导致的布局崩坏。
2.视觉层级 (Visual Hierarchy)： 通过字体大小、字重（Weight）、颜色深浅来区分信息的重要性。标题（H1-H3）与正文要有明显区分，引导用户视线。
3.一致性 (Consistency)： 按钮、输入框、卡片等组件在不同页面下的圆角（Radius）、阴影（Shadow）、间距（Padding/Margin）必须保持统一。
4.交互反馈 (Interaction)： 所有可点击元素（按钮、链接）必须有明确的悬停（:hover）和点击（:active）状态变化（如颜色变深、位移、阴影变化），拒绝“死”链接。
5.留白艺术 (Whitespace)： 拒绝密集堆砌。元素之间要保持足够的呼吸感，使用 margin 和 padding 来分隔内容块。
